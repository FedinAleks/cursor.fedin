Перелік основних команд

  • 1. git config --global user.name Aleksandr - ця команда вказує ваше імʼя
  • 2. git config --global user.email - вказує вашу пошту
  • 3. git init - ця команда створює новий локальний репозиторій Git
  • 4. git status - ця команда показує стан робочого дерева і індексу, показуючи, які файли змінені або додані, але ще не збережені в коміт.
  • 5. git add - ця команда додає змінені або нові файли в індекс для подальшого збереження в коміт.
  • 6. git commit - ця команда зберігає зміни у репозиторії
  • 7. git log - ця команда виводить історію комітів у репозиторії
  • 8. git checkout - ця команда дозволяє переміститися між гілками, створеними у репозиторії
  • 9. git branch - ця команда дозволяє переглядати список усіх гілок, створених у репозиторії
  • 10. git push - ця команда дозволяє відправити зміни з локального репозиторію на віддалений репозиторій
  • 11. git pull - ця команда дозволяє отримати зміни з віддаленого репозиторію та об'єднати їх з локальним репозиторієм.
  • 12. git merge - ця команда дозволяє об'єднати зміни з однієї гілки у іншу.
  • 13. git clone - ця команда дозволяє скопіювати віддалений репозиторій

    • Advanced

  • git revert - ця команда дозволяє скасувати зміни, які були зроблені в попередньому коміті. Команда створює новий коміт, який відміняє зміни, які були зроблені в попередньому коміті.
  • git reset - ця команда дозволяє скасувати зміни, які були зроблені в попередніх комітах. Команда змінює поточний коміт або гілку на певний коміт, видаляючи всі коміти, які були зроблені після нього. Ця команда може використовуватися для повернення до попереднього стану репозиторію.
  • git rebase - ця команда дозволяє перенести зміни з однієї гілки на іншу, зберігаючи при цьому історію комітів. Команда перебудовує історію комітів шляхом застосування комітів з однієї гілки до іншої.
  • git cherry-pick - ця команда дозволяє вибрати один або кілька комітів з однієї гілки і застосувати їх до іншої гілки. Команда створює новий коміт в приймаючій гілці, який містить зміни з вибраних комітів.
  • git stash - ця команда дозволяє зберегти незавершені зміни в стеку застібок і прибрати їх з поточної гілки, щоб працювати з іншими гілками або комітами. Команда дозволяє повернутися до збережених змін пізніше.

    • .gitignore

    .gitignore - це файл, який містить список файлів або каталогів, які Git повинен ігнорувати при виконанні команд. Це дозволяє виключити файли з контролю версій, які не повинні бути включені в репозиторій. В залежності від конкретного проекту та середовища розробки, .gitignore може містити різні типи файлів, такі як файли налаштувань, файли логів, файли змінних середовища, файли завантаження, файли з тимчасовими даними та інші. .gitignore файл зазвичай знаходиться в кореневій директорії репозиторію та містить список шаблонів файлів та каталогів, які Git повинен ігнорувати. Ці шаблони можуть включати підстановочні символи для відповідності файловим шаблонам.
